# Анализ пользовательского спроса на билеты Яндекс Афиши

Данный проект представляет собой исследовательский анализ данных о бронировании билетов сервиса Яндекс Афиша. Цель работы — изучить изменения в пользовательском спросе, поведение покупателей в зависимости от сезона, типа устройства и других факторов, а также проверить статистические гипотезы для выработки рекомендаций.

## 📋 Описание данных

Анализ проводился на основе трех датасетов:

1.  **`final_tickets_orders_df.csv`** — информация о заказах:
    *   `order_id`, `user_id` — идентификаторы заказа и пользователя.
    *   `created_dt_msk`, `created_ts_msk` — дата и время создания заказа (мск).
    *   `event_id` — идентификатор мероприятия.
    *   `revenue`, `total`, `tickets_count` — выручка, общая сумма заказа, количество билетов.
    *   `currency_code` — валюта оплаты (`rub`/`kzt`).
    *   `device_type_canonical` — тип устройства (`mobile`/`desktop`).
    *   `service_name` — билетный оператор.
    *   `days_since_prev` — количество дней с предыдущей покупки пользователя.
2.  **`final_tickets_events_df.csv`** — информация о мероприятиях:
    *   `event_id`, `event_name` — идентификатор и название мероприятия.
    *   `event_type_description`, `event_type_main` — тип и основная категория мероприятия (концерт, театр и т.д.).
    *   `region_name`, `city_name`, `venue_name` — место проведения.
3.  **`final_tickets_tenge_df.csv`** — курс тенге к рублю.
